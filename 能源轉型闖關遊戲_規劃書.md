# 能源轉型闖關互動小遊戲 — 程式設計規劃書

## 1. 專案目的

建立一個以「能源轉型」為主題的互動式闖關小遊戲，目標是讓大學生了解不同電廠的特性，並透過調整電廠數量，學習「發電量、建造成本、排碳量、環境風險」之間的權衡。

## 2. 使用者對象與學習目標

- 使用者對象：大學生
- 學習目標：
  - 認識火力、風力、水力、太陽能、核能五種發電方式
  - 了解不同電廠對發電能力、成本、碳排放與環境風險的影響
  - 透過互動操作理解「能源轉型」不是單一解，而是多種能源組合的平衡

## 3. 最小可運作版本（MVP）範圍

先實作以下功能：

1. 顯示五種電廠的發電量、建造成本、排碳量、環境風險
2. 玩家可對每種電廠進行加減數量
3. 系統即時計算總發電量、總成本、總排碳量、總環境風險
4. 提供一個簡單闖關判定：是否達成能源轉型挑戰目標
5. 支援手機、平板、電腦的 RWD 版面

## 4. 遊戲核心概念

### 4.1 遊戲目標
玩家在每一關中調整電廠數量，目標是：
- 提供足夠發電量
- 降低排碳量
- 降低環境風險
- 控制總建造成本

### 4.2 闖關方式
初版採用「單一關卡」形式：
- 玩家一開始看到預設電廠配置
- 可自由增減電廠數量
- 系統會即時更新數值
- 若達成挑戰條件，顯示「成功通關」；否則顯示「再試一次」提示

## 5. 電廠資料設計

所有電廠資料以一個陣列物件表示，後續由 JavaScript 動態渲染。

### 5.1 電廠初始數量
- 火力發電廠：8
- 風力發電廠：2
- 水力發電廠：2
- 太陽能發電廠：3
- 核能發電廠：0

### 5.2 建議的初始數值（可作為第一版平衡設定）

- 火力發電廠
  - 發電量：18
  - 建造成本：8
  - 排碳量：16
  - 環境風險：8

- 風力發電廠
  - 發電量：12
  - 建造成本：10
  - 排碳量：0
  - 環境風險：3

- 水力發電廠
  - 發電量：15
  - 建造成本：12
  - 排碳量：0
  - 環境風險：2

- 太陽能發電廠
  - 發電量：9
  - 建造成本：9
  - 排碳量：0
  - 環境風險：1

- 核能發電廠
  - 發電量：20
  - 建造成本：15
  - 排碳量：1
  - 環境風險：4

### 5.3 計算邏輯

每次調整數量後，重新計算：
- 總發電量 = 各電廠數量 × 發電量
- 總建造成本 = 各電廠數量 × 建造成本
- 總排碳量 = 各電廠數量 × 排碳量
- 總環境風險 = 各電廠數量 × 環境風險

## 6. 闖關判定條件

初版建議設定一個簡單且可達成的目標，例如：
- 總發電量至少 180
- 總排碳量小於 130
- 總環境風險小於 70
- 總建造成本不超過 220

當玩家達成這些條件時，顯示「成功通關」。

## 7. 使用者介面規劃

### 7.1 頁面結構
單一 HTML 頁面，包含以下區塊：
1. 頁首標題
   - 標題：能源轉型闖關遊戲
   - 簡短說明
2. 電廠卡片區
   - 每個電廠一張卡片
   - 卡片內顯示：名稱、數量、四個數值
   - 提供加號與減號按鈕
3. 結果摘要區
   - 顯示總發電量、總成本、總排碳量、總環境風險
4. 闖關狀態區
   - 顯示目前是否達成目標
   - 顯示成功/失敗訊息

### 7.2 卡片內容
每張卡片至少包含：
- 電廠名稱
- 目前數量
- 發電量
- 建造成本
- 排碳量
- 環境風險
- 加減按鈕

## 8. RWD 規劃

採用手機優先設計：
- 手機：單欄排版
- 平板：兩欄排版
- 桌機：三欄或四欄排版

建議使用：
- 彈性容器
- CSS Grid 或 Flexbox
- 介面元件尺寸隨視窗縮放

## 9. 技術規格

### 9.1 檔案結構
建議只使用單一 HTML 檔案：
- index.html

其中內含：
- HTML 結構
- CSS 樣式
- JavaScript 邏輯

### 9.2 技術限制
- 不使用任何外部函式庫
- 不使用框架
- 以原生 HTML、CSS、JavaScript 實作
- 繁體中文介面

## 10. JavaScript 功能規格

### 10.1 建議函式與責任分離
- init()
  - 初始化電廠資料
  - 渲染卡片與統計資料
- renderPlants()
  - 動態生成電廠卡片
- updateSummary()
  - 更新結果摘要與闖關狀態
- changePlantCount(id, delta)
  - 變更指定電廠數量
- checkGoal()
  - 判斷是否達成挑戰條件

### 10.2 事件處理
- 點擊加號：數量 +1
- 點擊減號：數量 -1
- 數量不可小於 0
- 更新畫面後立即重新計算總值

## 11. UI/UX 建議

- 使用清楚的顏色區分：
  - 綠色：低碳、低風險
  - 橙色：中等風險
  - 紅色：高碳或高風險
- 數值使用明確標籤，避免混淆
- 每個卡片都要有明確的互動反饋
- 結果摘要可即時更新，讓玩家有立即回饋

## 12. 實作順序建議

1. 建立 HTML 基礎結構
2. 製作 CSS 版型與 RWD
3. 建立電廠資料與初始狀態
4. 實作卡片渲染
5. 實作加減數量邏輯
6. 實作統計計算與挑戰判定
7. 加入成功/失敗提示與視覺回饋
8. 進行手機、平板、電腦的尺寸測試

## 13. 完成標準

以下條件達成時，視為 MVP 完成：
- 可正常顯示五種電廠資料
- 可加減各電廠數量
- 會即時更新四個總數值
- 會顯示闖關結果
- 可於手機、平板、電腦正常使用

## 14. 實作備註

- 這個版本以「能跑、能互動、能學習」為優先，不追求高度複雜的遊戲系統
- 未來可擴充為多關卡、分數系統、能源組合提示與動畫效果
